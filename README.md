# A Share Stock Picker

## 介绍

`a-share-stock-picker` 是一个面向 Codex 的 A 股选股 skill，支持两种核心工作模式：

- `盘后 / 盘前模式`：适用于收盘后到次日开盘前的观察池整理、短中长线候选分析和条件交易计划输出
- `T+1 尾盘建仓模式`：适用于交易日尾盘建仓、次日按预设计划卖出，解决 A 股 `T+1` 规则下“当天推荐当天卖不掉”的问题

这个仓库提供的不是单纯提示词，而是一套完整的 skill 包，包含：

- `SKILL.md`：核心行为定义
- `references/`：选股框架、价格规则、输出模板、尾盘模式说明和使用说明
- `scripts/`：行情抓取、候选打分、指标计算、催化抓取、尾盘快照和报告生成脚本
- `agents/`：agent 配置

## 适用场景

- `推荐股票`
- `盘前选股`
- `按收盘后信息做观察池`
- `给我短中长线标的`
- `给我条件交易计划`
- `尾盘选股`
- `尾盘买入、次日卖出`
- `按 A 股 T+1 规则做短线计划`

## 核心特点

- 仅覆盖中国大陆 A 股
- 默认支持短线、中线、长线三个周期
- 使用可验证的行情数据，不依赖用户手填价格
- 优先输出 `观察池 / 候选名单 / 条件交易计划`
- 数据验证不足时，不强行给出绝对化买卖建议
- 新增 `T+1 尾盘建仓模式`，让短线建议更贴近 A 股实盘约束

## 工作模式

### 1. 盘后 / 盘前模式

适合：

- 收盘后整理次日观察池
- 盘前输出短中长线候选
- 基于上个交易日和隔夜信息做条件交易计划

特点：

- 使用最近一个完整交易日作为主要价格锚点
- 若当前时间在交易日 `15:00` 之前，则锚点仍是前一个交易日；若已经过了交易日 `15:00`，则锚点切换为当日收盘，对应下一个交易日盘前口径
- 结合近 5 日、20-60 日、6-12 个月历史窗口判断结构
- 默认输出短线、中线、长线各 `5` 只
- 更适合大多数常规盘前分析请求

### 2. T+1 尾盘建仓模式

适合：

- 想在 `D` 日尾盘买入
- 想在 `D+1` 日开盘后或盘中卖出
- 需要规避 A 股 `T+1` 带来的“同日推荐买卖冲突”

特点：

- 在交易日最后一小时刷新分时数据
- 结合 `14:00` 后走势、尾盘量能、收盘位置和历史结构重新打分
- 输出两段式计划：
  - `今日尾盘建仓计划`
  - `次日卖出计划`

## 内置能力

### 数据与分析

- 先从同花顺全市场排行里预筛约 `200` 只高价值股票，再进入详细分析
- 拉取 A 股行情与历史数据
- 计算均线、ATR、MACD 线索和量比等技术指标
- 把资金流入、资金流出、资金净流入、成交额和换手率一起纳入分析，而不只停留在预筛阶段
- 主力资金流向会直接影响推荐优先级、核心逻辑和放弃条件，而不是只做展示说明
- 从同花顺多页面、多入口获取个股催化、公告、研报、新闻摘要和上下文信息
- 对同花顺上下文做有用性排序，并优先保留新鲜新闻
- 输出时不仅给股票，还要求结合板块/行业、新闻、政策和披露做分析
- 无法验证的数据不补写、不捏造，缺失时会明确说明
- 将候选按短线、中线、长线进行打分和分桶
- 提取尾盘分时快照，识别 `14:00` 后回流和尾盘强度

### 输出与表达

- 生成中文 Markdown 选股报告
- 支持结构化表格输出
- 标准报告使用固定模板和固定列顺序，默认盘前模式与 T+1 模式都不应随意变形
- 优先使用条件触发式交易语言
- 在尾盘模式下，明确拆分建仓日和卖出日

## 使用方式

### 安装到 Codex

```bash
cp -R a-share-stock-picker ~/.codex/skills/
```

### 调用示例

#### 盘后 / 盘前模式

```text
使用 $a-share-stock-picker 给我一份盘前 A 股观察池，短线、中线、长线各 5 只。
```

```text
使用 $a-share-stock-picker 基于收盘后信息，输出均衡型候选名单和条件交易计划。
```

#### T+1 尾盘建仓模式

```text
使用 $a-share-stock-picker 按 T+1 尾盘建仓模式，给我一份今日尾盘可买、次日可卖的 A 股短线计划。
```

```text
使用 $a-share-stock-picker 在 14:30 之后重新分析，输出今日尾盘建仓计划和次日卖出计划。
```

### T+1 固定提示词模板

#### `14:10` 预备版

```text
使用 $a-share-stock-picker 按 T+1 尾盘建仓模式，先做一版预备观察池。
请基于当前盘中信息，输出今日尾盘可买、次日可卖的 A 股短线计划，先给我 5 个候选，并标出优先级。
```

#### `14:35` 最终版

```text
使用 $a-share-stock-picker 在 14:30 之后重新分析。
请输出今日尾盘建仓计划和次日卖出计划，按 T+1 模式给我 5 个最终候选，并按可执行优先级排序。
```

#### `D+1 09:25` 卖出版

```text
使用 $a-share-stock-picker 复核我昨天的 T+1 尾盘建仓计划。
请结合今日竞价和开盘前信息，输出这 5 只的今日卖出优先级、止盈位、止损位和放弃条件。
```

说明：

- 第一条用于预筛，不等于最终下单清单
- 第二条是尾盘执行前最重要的一次刷新
- 第三条已经是 `D+1`，这里应输出 `今日卖出计划`，不再写 `明日卖出`
- 这三条可以单独使用，但更推荐按时间顺序联动使用

### 推荐工作方式

#### 盘后 / 盘前模式

- 先做全市场前 `200` 只股票预筛
- 再抓取行情和历史数据
- 再结合板块/行业、新闻、政策和披露信息
- 最后输出观察池或条件交易计划

#### T+1 尾盘建仓模式

- `14:00-14:20` 拉取当日分时和历史结构
- `14:20-14:40` 做尾盘强度筛选
- `14:45-14:57` 输出最终建仓计划
- `D+1 09:20-09:30` 复核竞价和开盘前信息，输出今日卖出计划
- `D+1 09:30-10:30` 根据计划执行卖出

如果要给出精确买点、止损和目标位，应确保底层价格数据是最新且已验证的。

## 注意事项

- 盘后 / 盘前模式最适合收盘后到次日开盘前
- 默认盘前模式目标输出为 `5+5+5`
- 尾盘模式更适合 A 股短线实盘 `T+1`
- 不建议在盘中数据未校验时直接给出绝对化结论
- 如果证据不足，应退回观察池或条件计划表达
